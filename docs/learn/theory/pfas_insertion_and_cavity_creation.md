# PFAS Molecule Insertion and Cavity Creation in LAMMPS

This document explains the technical methodology used in our LAMMPS simulation scripts (such as `pfoa_in_water.lammps`) to insert a complex PFAS molecule into a dense water slab and safely resolve the resulting atomic overlaps without crashing the simulation.

## 1. Where and How the PFAS is Inserted

The PFAS molecule is inserted exactly in the center of the pre-equilibrated water slab using the `read_data` command with the `add` keyword:

```lammps
read_data data/PFOA_stripped.data add append offset 2 1 1 0 0 shift 18.636 18.636 18.5
```

Here is a breakdown of what this command is doing:
*   `read_data [...] add append`: This tells LAMMPS to append the atoms, bonds, angles, etc., from `PFOA_stripped.data` to the existing water system without wiping the current system state.
*   `offset 2 1 1 0 0`: Because the initial water data file (`1H2O.data`) already uses atom types 1 and 2 (O and H), and bond type 1, etc., this offset bumps up the incoming PFOA atom types so they start at type 3, bond types at type 2, etc. This dynamically avoids type numbering conflicts.
*   `shift 18.636 18.636 18.5`: This physically translates the PFOA molecule coordinates to the center of the constructed water box:
    *   `x = 18.636` (Half of the typical replicated $12 \times 3.106 \approx 37.27$ Å box)
    *   `y = 18.636` (Same as x)
    *   `z = 18.5` (Half of the roughly 37 Å thick water slab).

By placing the PFAS exactly in the center of the slab, it is fully solvated and acts as an objective starting point to measure its migration to the air-water interface over time.

## 2. How the Cavity is Created

Because the PFOA is forcibly inserted right into the dense center of the water slab, its atoms will severely overlap with the existing water molecules. If this system were run immediately with standard long-range solvers, the nearly infinite repulsive forces would cause the simulation to crash instantly.

The cavity to accommodate the PFOA is carved out dynamically during **Phase 1: Cavity Healing**. It is achieved safely in two main steps:

### Step A: Carving the Rough Cavity
LAMMPS is instructed to delete overlapping molecules using the `delete_atoms` command:

```lammps
delete_atoms overlap 3.0 water pfoa mol yes
```

LAMMPS scans the system and deletes any water molecule that is within 3.0 Ångströms of any atom in the inserted PFOA molecule. 
*   The `mol yes` flag is critical: it ensures the *entire* $H_2O$ molecule is removed, maintaining the correct mass and charge stoichiometry of the water, rather than just deleting a single overlapping Hydrogen or Oxygen atom. 
*   This immediately punches a hole (a cavity) in the dense water slab that perfectly matches the shape of the inserted PFOA molecule.

### Step B: The "Healing" Run
Even after deleting waters within 3.0 Å, the remaining water molecules on the edge of that boundary are still in very harsh contact with the PFOA (e.g., atoms at 3.1 Å that repel each other strongly). To resolve this smoothly, a specialized "healing" run is performed:

1.  **Short-Range Electrostatics Only:** A special pair style is used (`pair_style lj/cut/coul/cut 10.0`) instead of the accurate long-range PPPM solver. The PPPM iterative solver diverges and crashes when given the extreme charge densities created by these remaining tight contacts.
2.  **Energy Bleeding:** The system is run for a very short duration at a tiny time step (e.g., 0.1 fs, instead of the normal 1.0 or 2.0 fs) using a strong Langevin thermostat (`fix pull_heat all langevin...`). This setup aggressively bleeds off the massive kinetic energy generated by the atoms forcefully pushing away from each other.

After this short (typically 0.5 picosecond) "healing" run, the newly formed cavity has safely relaxed and expanded slightly around the PFOA molecule without blowing the system apart. The system is then stable enough to re-enable normal long-range electrostatics (PPPM) and larger timesteps for the production run.

## 3. Physical Estimates of the Cavity (PFOA)

Based on the 3D conformer generated for PFOA:
* **Molecule Length:** The backbone extends roughly $10.4$ Å end-to-end. Including the van der Waals radii of the terminal atoms, the effective length of the PFOA molecule is approximately **13 – 14 Ångströms**.
* **Removed Waters:** The `delete_atoms overlap 3.0` command carves out a roughly cylindrical volume around the PFOA chain. Based on bulk water density ($0.033 \text{ molecules/\AA}^3$) and this cavity volume, approximately **20 to 25 water molecules** are removed to accommodate the single PFOA molecule.
